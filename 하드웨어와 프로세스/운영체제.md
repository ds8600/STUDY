# 운영체제(OS)

## 1. 한 줄 정의
- 운영체제는 **[하드웨어 자원을 관리하고]**, **[프로그램이 실행될 수 있게 해주는]** 기본 소프트웨어다.

## 2. 운영체제가 필요한 이유
- 프로그램이 CPU/메모리/디스크 같은 자원을 직접 만지면 **충돌/오류**가 쉽게 발생한다.
- 운영체제가 중간에서 **규칙(추상화/보호)**을 만들어서 안정적으로 쓰게 해준다.

## 3. 운영체제의 핵심 역할 3가지
### (1) 자원 관리(Resource Management)
- CPU 시간, 메모리 공간, 저장장치, 입출력 장치 등을 **누가/언제/얼마나** 쓸지 관리한다.

### (2) 프로세스 관리(Process Management)
- 프로그램 실행 단위를 **프로세스**로 만들어 실행/중지/전환을 관리한다.
- 여러 프로세스를 동시에 돌리는 것처럼 보이게 만든다(시분할).

### (3) 사용자/프로그램 인터페이스 제공
- 사용자에게는 **GUI/CLI**를 제공하고,
- 프로그램에게는 **시스템 콜(System Call)**로 기능을 제공한다.

## 4. 커널(Kernel)과 운영체제
- **커널**: 운영체제의 핵심(가장 중요한 부분)
- 커널이 하는 일: CPU 스케줄링, 메모리 관리, 파일 시스템 관리, 장치 제어 등

## 5. 시스템 콜(System Call) 예시
- 프로그램이 “운영체제 기능”을 쓰려면 시스템 콜을 통해 요청한다.
- 예: 파일 읽기/쓰기, 네트워크 통신, 프로세스 생성 등
