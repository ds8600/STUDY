# 인터럽트(Interrupt)

## 1. 한 줄 정의
- 인터럽트는 **CPU가 하던 일을 잠깐 멈추고**, 중요한 이벤트를 처리하게 하는 신호다.

## 2. 인터럽트가 필요한 이유
- CPU가 I/O를 계속 기다리면 비효율적이다.
- 인터럽트로 “필요할 때만” 처리해서 CPU를 더 효율적으로 쓴다.

## 3. 종류
- 하드웨어 인터럽트: 키보드 입력, 디스크 작업 완료, 네트워크 패킷 도착 등
- 소프트웨어 인터럽트: 시스템 콜 같은 소프트웨어 요청

## 4. 처리 흐름(간단)
1) 인터럽트 발생
2) CPU가 현재 상태 저장(레지스터 등)
3) 인터럽트 서비스 루틴(ISR) 실행
4) 원래 작업으로 복귀
