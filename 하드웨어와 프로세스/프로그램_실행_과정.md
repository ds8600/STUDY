# 프로그램 실행 과정

## 1. 실행 흐름(큰 그림)
1) 저장장치(SSD/HDD)에 있는 실행 파일(프로그램)을 실행 요청
2) 운영체제가 프로그램을 **메모리(RAM)**에 로드
3) 프로세스 생성(프로세스 ID 부여, 자원 할당)
4) CPU가 명령어를 실행(스케줄링에 따라 실행 시간 배정)

## 2. 로드(Loading)란?
- 프로그램의 코드/데이터를 RAM에 올리는 과정
- 실행 중 필요한 라이브러리/데이터도 함께 연결될 수 있음

## 3. 프로세스 관점에서 메모리 구성(예시)
- Code(Text): 실행 코드
- Data: 전역/정적 변수
- Heap: 동적 할당
- Stack: 함수 호출/지역 변수

## 4. 내가 이해한 핵심
- “프로그램(파일)이 실행되면 운영체제가 프로세스를 만들고, CPU는 그 프로세스를 번갈아 실행한다.”
